Phase 1: Unblock the LLM Client (Critical Prerequisite)

Before building any agents, fix the synchronous bottleneck in your system.

Why?
An agent will make 3–5 API calls before returning a result. If your LLM client blocks the event loop, the entire application will freeze.

Instructions

Open:
backend-python/app/LLM/llm_client.py

Replace the synchronous requests.post inside call_llama() with:

httpx.AsyncClient

Update all controller functions that use llm_client.py to use await.

Phase 2: Install the Agent Framework

We will use LangGraph because it is designed for cyclical, stateful agent loops in Python.

Instructions

In the backend-python directory:

pip install langgraph langchain-core httpx

Phase 3: Architect the Docker Agent Loop

Currently, docker_deploy_agent.py uses a large system prompt and generates everything in one shot.
We will convert this into a state machine.

1. Define the State

Create a new file:

app/LLM/docker_graph.py


Define a TypedDict called AgentState:

project_id: str

metadata: dict

current_dockerfile: str

build_output: str (logs/errors)

retry_count: int

2. Create the Tools

Import docker_service.py

Wrap build_project_stream() into a function that:

Executes the build

Returns stderr if the build fails

Do not stream to frontend

3. Define the Nodes (Actions)

Node 1: generate_files

Calls Ollama using metadata

Updates current_dockerfile in state

Node 2: test_build

Runs Docker build tool

If failed:

Save error in build_output

Increment retry_count

4. Define the Edges (Logic)

Flow:

generate_files → test_build

Conditional from test_build:

If build succeeds → END

If build fails and retry_count < 3
→ loop back to generate_files (pass error log)

If build fails and retry_count == 3
→ END (return failure)

5. Update the Controller

File:

app/controllers/docker_controller.py


Update docker_chat_handler

Trigger the new LangGraph workflow

Remove static call_llama_stream

Phase 4: Architect the Terraform Agent (Human-in-the-Loop)

Current issue:
terraform_deploy_agent.py generates and applies main.tf blindly.

We will create an agent that:

Plans infrastructure

Fixes errors

Waits for user approval

1. Define the State

Create:

app/LLM/terraform_graph.py


State fields:

main_tf_content

plan_output

is_approved

2. Create the Tools

Import aws_service.py and expose:

write_terraform

terraform_init

New tool: terraform_plan

3. Define Nodes & Edges

Node 1: generate_hcl

LLM generates main.tf

Node 2: run_plan

Executes terraform plan

Conditional Logic

If plan fails → loop back to generate_hcl with error

If plan succeeds → proceed

Node 3: wait_for_human

LangGraph interrupt

Sends clean plan output to frontend

Node 4: apply_infrastructure

Runs aws_service.terraform_apply()

Triggered only after user clicks Approve

Phase 5: Update Frontend Streaming Logic

Current behavior:

React (DeployPage.tsx) expects raw text tokens via SSE.

Problem:
Agents run loops → raw tokens will show multiple rewrites (confusing UX).

Backend Update

Modify FastAPI SSE to send JSON status updates instead of text:

Example:

yield 'data: {"step": "test_build", "status": "failed", "retry": 1}\n\n'

Frontend Update

In AIChatSidebar:

Parse JSON SSE events

Display as a progress checklist, e.g.:

□ Generating Files
□ Testing Build
□ Pushing to Hub


Instead of chat-style dialogue.
